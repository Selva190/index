What is the Agent Development Kit?

Google's Agent Development Kit (ADK) is an open-source Python framework for building, orchestrating, and deploying AI agents. It provides first-class support for Gemini models and integrates seamlessly with Google Cloud.

🧠 Model Agnostic
Use Gemini 2.0 Flash, Pro, or any Vertex AI model. Switch models with one line of config.
🔧 Built-in Tool Use
Define Python functions as tools. ADK handles function calling, schemas, and execution loop automatically.
🤝 Multi-Agent
Compose agents into hierarchies. Route tasks to specialist sub-agents using orchestrators.
🌐 Deployment Ready
Built-in FastAPI server, session management, and Vertex AI Agent Engine integration.
💡 ADK vs LangChain: ADK is optimized for Gemini function calling and Google Cloud. LangChain is more general-purpose. Use ADK when you're building on GCP with Gemini.

Define Agent Tools

Tools are Python functions with type hints and docstrings. ADK uses these to generate the function-calling schema sent to Gemini.

python tools/custom_tools.py 
import httpx
from typing import Optional
from google.adk.tools import ToolContext


def search_web(query: str, max_results: int = 5) -> dict:
    """Search the web for information on a given topic.
    
    Args:
        query: The search query string.
        max_results: Maximum number of results to return (1-10).
    
    Returns:
        Dictionary with 'results' list and 'total' count.
    """
    # Replace with your actual search API (Serper, SerpAPI, etc.)
    response = httpx.get(
        "https://api.search-service.com/search",
        params={"q": query, "num": max_results},
        headers={"X-API-Key": "YOUR_KEY"}
    )
    data = response.json()
    return {
        "results": data["organic_results"],
        "total": len(data["organic_results"])
    }


def get_weather(city: str) -> dict:
    """Get current weather for a city.
    
    Args:
        city: City name (e.g., 'San Francisco, CA').
    
    Returns:
        Weather data including temperature, conditions, humidity.
    """
    response = httpx.get(
        f"https://api.openweathermap.org/data/2.5/weather",
        params={"q": city, "appid": "YOUR_OWM_KEY", "units": "metric"}
    )
    data = response.json()
    return {
        "city": city,
        "temperature_c": data["main"]["temp"],
        "conditions": data["weather"][0]["description"],
        "humidity_pct": data["main"]["humidity"]
    }


def save_to_database(
    collection: str,
    document_id: str,
    data: dict,
    ctx: ToolContext          # Injected automatically by ADK
) -> dict:
    """Persist data to Firestore.
    
    Args:
        collection: Firestore collection name.
        document_id: Document identifier.
        data: Dictionary of fields to save.
    
    Returns:
        Confirmation with document path.
    """
    # Access session state via ToolContext
    user_id = ctx.state.get("user_id", "anonymous")
    doc_ref = ctx.firestore.collection(collection).document(document_id)
    doc_ref.set({**data, "saved_by": user_id})
    return {"status": "saved", "path": f"{collection}/{document_id}"}
💡 Tip: Write detailed docstrings — Gemini uses them to decide when and how to call your tool. Vague descriptions lead to poor tool selection.

Multi-Agent Orchestration

Build complex pipelines by composing specialist agents under an orchestrator.

python agents/multi_agent.py
from google.adk.agents import LlmAgent, SequentialAgent, ParallelAgent # Specialist agents researcher = LlmAgent( name="researcher", model="gemini-2.0-flash-001", instruction="You are a web researcher. Find and summarize relevant information.", tools=[search_web], output_key="research_results", ) writer = LlmAgent( name="writer", model="gemini-2.0-flash-001", instruction="""You are a technical writer. Research results: {research_results} Transform the research into clear, engaging content.""", output_key="draft_content", ) reviewer = LlmAgent( name="reviewer", model="gemini-2.0-flash-001", instruction="""You are a critical reviewer. Draft content: {draft_content} Review for accuracy, clarity, and completeness.""", output_key="final_content", ) # Sequential pipeline: Research → Write → Review content_pipeline = SequentialAgent( name="content_pipeline", description="Research, write, and review content end-to-end.", sub_agents=[researcher, writer, reviewer], ) # Parallel execution: run multiple agents simultaneously multi_search = ParallelAgent( name="multi_search", description="Search multiple sources in parallel.", sub_agents=[researcher, researcher], # Different prompts each )
ℹ️ State sharing: Use output_key to pass data between sequential agents. ADK stores it in session state and injects it via template variables in instructions.

Containerize with Docker

python main.py — FastAPI server 
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService
from google.adk.types import Content, Part
from agents.root_agent import root_agent
import uvicorn

app = FastAPI(title="AI Agent API", version="1.0.0")
session_service = InMemorySessionService()
runner = Runner(agent=root_agent, session_service=session_service)

class ChatRequest(BaseModel):
    message: str
    session_id: str | None = None
    user_id: str = "default-user"

class ChatResponse(BaseModel):
    response: str
    session_id: str

@app.get("/health")
async def health_check():
    return {"status": "healthy"}

@app.post("/chat", response_model=ChatResponse)
async def chat(request: ChatRequest):
    try:
        # Create or reuse session
        if not request.session_id:
            session = await session_service.create_session(
                app_name="my-agent-app",
                user_id=request.user_id
            )
            session_id = session.id
        else:
            session_id = request.session_id

        # Run the agent
        final_response = ""
        async for event in runner.run_async(
            user_id=request.user_id,
            session_id=session_id,
            new_message=Content(parts=[Part(text=request.message)])
        ):
            if event.is_final_response() and event.content:
                for part in event.content.parts:
                    if part.text:
                        final_response += part.text

        return ChatResponse(response=final_response, session_id=session_id)
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8080)

Deploy to Cloud Run

bash deploy to Cloud Run 
gcloud run deploy my-ai-agent \
  --image=$IMAGE_URL \
  --platform=managed \
  --region=$REGION \
  --allow-unauthenticated \
  --min-instances=0 \
  --max-instances=10 \
  --memory=2Gi \
  --cpu=2 \
  --timeout=300 \
  --concurrency=80 \
  --set-env-vars="GOOGLE_CLOUD_PROJECT=${PROJECT_ID}" \
  --set-env-vars="GOOGLE_CLOUD_LOCATION=${REGION}" \
  --set-env-vars="MODEL_NAME=gemini-2.0-flash-001" \
  --service-account=agent-sa@${PROJECT_ID}.iam.gserviceaccount.com
⚡ Concurrency
Set --concurrency=80 for async agents. Reduce to 1-10 for CPU-heavy workloads or stateful in-memory sessions.
🔐 Service Account
Create a dedicated SA with roles: aiplatform.user, datastore.user. Never use default compute SA in production.
💾 Memory
Start with 2Gi. Increase if using large context windows or processing documents. Monitor with Cloud Monitoring.
⏱️ Timeout
Set to 300s (5 min) for agent workloads. Multi-step agents with many tool calls can take 30-120 seconds.
⚠️ Cold starts: With --min-instances=0, expect 3-8s cold starts. Use --min-instances=1 for latency-sensitive production workloads (increases cost ~$15-30/month).

Monitoring & Observability

python middleware/telemetry.py 
from google.cloud import logging, monitoring_v3
from fastapi import Request
import time, structlog

# Structured JSON logging (Cloud Logging compatible)
log = structlog.get_logger()

async def telemetry_middleware(request: Request, call_next):
    start = time.monotonic()
    response = await call_next(request)
    duration = time.monotonic() - start
    
    log.info(
        "request_completed",
        path=request.url.path,
        method=request.method,
        status_code=response.status_code,
        duration_ms=round(duration * 1000, 2),
        session_id=request.headers.get("X-Session-ID"),
    )
    return response

# Add to FastAPI app
# app.middleware("http")(telemetry_middleware)

Patterns & Best Practices

🔄 Retry with exponential backoff
Gemini API has rate limits. Wrap tool calls and model calls with tenacity retry logic. Start with 1s delay, max 60s, max 5 retries.
🗄️ Use Firestore for persistent sessions
InMemorySessionService is lost on container restart. Use VertexAiSessionService or Firestore-backed sessions for production multi-turn conversations.
🔑 Secret Manager for API keys
Never store API keys in environment variables or code. Use Google Secret Manager and mount secrets to Cloud Run at runtime.
⚡ Gemini model selection
Flash for speed-critical tasks (tool routing, simple Q&A). Pro for complex reasoning, long documents, and multi-step planning. Use Flash as orchestrator, Pro for specialist agents.
🧪 Test agents deterministically
Mock LLM responses in unit tests with a MockLlm that returns predetermined tool calls and text. Test agent routing logic independently from model behavior.
🚀 Production checklist: IAM least-privilege · Secret Manager for secrets · Firestore sessions · min-instances=1 · Cloud Armor WAF · VPC connector for private resources · Cloud Trace enabled
